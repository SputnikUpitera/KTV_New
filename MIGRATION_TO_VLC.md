# Миграция на VLC и совместимость со старой системой

## ✅ Выполненные изменения:

### 1. **Порт API изменён: 9999 → 8888**
- Старая система использует порт 9999 (или может использовать)
- Наша система теперь на порту 8888
- **Конфликты исключены**

### 2. **Плеер: MPV → VLC**
- `remote_player/player.py` переписан для VLC
- Используется `/usr/bin/vlc` по умолчанию
- Поддержка X11 DISPLAY (:0)
- Совместим со старой системой

### 3. **Структура файлов: /opt/ktv/media/ → ~/MM/DD/HH-MM/**
- Конфиг обновлён: `media_base_path: ~`
- Совместимость со старой структурой
- Файлы будут создаваться в: `~/02/09/14-30/` (месяц/день/час-минута)
- Clips в: `~/clips/`

## 🔄 Следующие шаги (в процессе):

### 4. **Обновление daemon для новой структуры**
- Daemon должен создавать MM/DD/HH-MM/ директории
- Scheduler должен работать с этой структурой

### 5. **Настройка Samba для доступа из Windows**
- Шара `\\неттоп\home` для доступа к ~/
- Возможность просмотра/редактирования файлов из проводника Windows

### 6. **GUI обновления**
- Загрузка файлов в MM/DD/HH-MM/
- Кнопка "Открыть папку" → открывает `\\неттоп\home\`

## 🔀 Переключение систем:

**Использовать старую:**
```bash
crontab -e
# Раскомментировать: * * * * * ~/vlc_cron.sh
sudo systemctl stop ktv-daemon
```

**Использовать новую:**
```bash
crontab -e
# Закомментировать: # * * * * * ~/vlc_cron.sh
sudo systemctl start ktv-daemon
```

## 📂 Доступ к файлам из Windows:

После настройки Samba:
```
\\192.168.1.45\home\
  ├── 02/
  │   ├── 09/
  │   │   ├── 14-30/
  │   │   │   └── фильм.mp4
  │   │   └── 18-00/
  │   │       └── другой_фильм.mp4
  │   └── 10/
  ├── clips/
  │   ├── клип1.mp4
  │   └── клип2.mp4
```

Можно открыть в проводнике и работать как с локальными файлами!

## ⚠️ Важно:

1. **Не запускайте обе системы одновременно** - они будут бороться за VLC
2. **Файлы общие** - изменения в проводнике видны обеим системам
3. **Backup** - перед полным переходом протестируйте новую систему
