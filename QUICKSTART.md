# Быстрый старт OperatorKTV

## За 5 минут до первого запуска

### 1. Установка на Windows (1 минута)

```cmd
# Перейти в папку проекта
cd KTV_New

# Установить зависимости
pip install -r requirements_windows.txt

# Собрать offline пакет
python build_offline_package.py
```

### 2. Подготовка Linux (1 минута)

На Linux системе должен быть:
- SSH сервер
- Sudo права для пользователя

Проверьте SSH доступ:
```cmd
ssh user@192.168.1.100
```

### 3. Запуск и подключение (3 минуты)

```cmd
# Запустить клиент
python operator_ktv/main.py
```

В появившемся окне:
1. Введите IP, логин, пароль Linux системы
2. Нажмите **Подключиться**
3. Согласитесь на установку ПО
4. Подождите 2-3 минуты

Готово! Система установлена и готова к работе.

## Первое использование

### Добавить фильм в расписание

1. Вкладка **Фильмы**
2. Раскройте нужный месяц и найдите день
3. Перетащите видеофайл на день
4. Выберите время (например, 19:00)
5. Нажмите **Подтвердить**

Файл загрузится и будет воспроизведён в указанное время.

### Создать плейлист для фона

1. Вкладка **Клипы**
2. **Создать плейлист** → введите название
3. Перетащите видеофайлы в правую область
4. Дважды кликните по плейлисту для активации

Плейлист будет воспроизводиться когда нет запланированных файлов.

### Открыть консоль

**Инструменты → SSH Консоль**

Введите команды Linux, например:
```bash
ls /opt/ktv/media/
systemctl status ktv-daemon
```

## Частые вопросы

**Q: Когда начнётся воспроизведение?**  
A: Автоматически с 6:00 до 22:00. Настраивается в `/etc/ktv/config.json` на Linux.

**Q: Что если два файла на одно время?**  
A: Воспроизведутся оба по очереди.

**Q: Можно ли удалить файл после воспроизведения?**  
A: Нет, файл остаётся на системе. Удалите вручную через контекстное меню в расписании.

**Q: Какие форматы поддерживаются?**  
A: MP4, AVI, MKV, WEBM, MOV, FLV - всё что воспроизводит MPV.

**Q: Как остановить воспроизведение?**  
A: Отключите файл в расписании (двойной клик) или остановите daemon через SSH консоль.

## Полезные команды Linux

```bash
# Статус
sudo systemctl status ktv-daemon

# Перезапуск
sudo systemctl restart ktv-daemon

# Логи в реальном времени
sudo journalctl -u ktv-daemon -f

# Список файлов
ls -lh /opt/ktv/media/movies/
ls -lh /opt/ktv/media/clips/
```

## Поддержка

При проблемах проверьте:
1. SSH подключение: `ssh user@ip`
2. Daemon запущен: `systemctl status ktv-daemon`
3. Логи: `tail -f /var/log/ktv/daemon.log`
