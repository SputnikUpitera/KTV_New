# Следующие шаги - Миграция на VLC завершена

## ✅ Что УЖЕ сделано:

1. ✅ **API порт изменён: 9999 → 8888** (не конфликтует со старой системой)
2. ✅ **Player переписан: MPV → VLC** 
3. ✅ **Структура файлов: ~/MM/DD/HH-MM/** (как в старой системе!)
4. ✅ **Конфигурация обновлена** (vlc_path, display, новые пути)
5. ✅ **Samba подготовлен** для доступа из Windows

## 🔄 Что нужно СДЕЛАТЬ:

### 1. Пересоберите пакет (ОБЯЗАТЕЛЬНО!)

```cmd
python build_offline_package.py
```

Это создаст новый `offline_package/ktv_offline_package.tar.gz` со всеми изменениями.

### 2. На неттопе: Временно остановите старую систему

```bash
# Отключите старый cron (временно!)
crontab -e
# Закомментируйте строку: # * * * * * ~/vlc_cron.sh
# Сохраните и выйдите

# Убедитесь что VLC не запущен
killall vlc
```

### 3. Переустановите через OperatorKTV

```cmd
python operator_ktv/main.py
```

- Подключитесь к неттопу
- Меню → Инструменты → **Установить ПО**
- Подождите завершения установки

### 4. Настройте Samba (для доступа из Windows)

На неттопе выполните:

```bash
# Установите Samba если нет
sudo apt install -y samba

# Добавьте шару для home
sudo nano /etc/samba/smb.conf
```

Добавьте в конец файла:

```ini
[home]
   comment = KTV Media Files
   path = /home/user
   browseable = yes
   read only = no
   create mask = 0775
   directory mask = 0775
   valid users = user
```

Затем:

```bash
# Установите пароль Samba
sudo smbpasswd -a user
# Введите пароль (можно тот же что для SSH)

# Перезапустите Samba
sudo systemctl restart smbd
sudo systemctl enable smbd
```

### 5. Проверьте доступ из Windows

Откройте проводник Windows и введите:

```
\\192.168.1.45\home
```

Вы должны увидеть:
```
home/
├── clips/         ← здесь клипы
├── 02/           ← месяц
│   ├── 09/       ← день
│   │   ├── 14-30/  ← час-минута
│   │   └── 18-00/
│   └── 10/
```

### 6. Тестирование

1. **Загрузите тестовый файл:**
   - В OperatorKTV перетащите видеофайл
   - Выберите время (например, через 2 минуты)
   - Проверьте что файл появился в `\\192.168.1.45\home\MM\DD\HH-MM\`

2. **Проверьте что файл воспроизвёлся:**
   - Подождите указанное время
   - Меню → Инструменты → **Логи daemon**
   - Должна быть строка: "Starting VLC playback: ..."

### 7. Если всё работает - можно удалить старый cron

```bash
crontab -e
# Полностью удалите строку с vlc_cron.sh
```

## 🔄 Откат на старую систему (если нужно)

Если что-то не работает, легко вернуться:

```bash
# Остановить новый daemon
sudo systemctl stop ktv-daemon
sudo systemctl disable ktv-daemon

# Включить старый cron
crontab -e
# Раскомментировать: * * * * * ~/vlc_cron.sh

# Всё! Старая система снова работает
```

Файлы остаются на месте - обе системы их видят!

## ⚠️ Важные моменты:

1. **Не запускайте обе системы одновременно!** Они будут бороться за VLC.
2. **Порты разные:** Старая не использует порт (cron), новая на 8888.
3. **Файлы общие:** Изменения видны обеим системам.
4. **VLC один:** Обе системы используют `/usr/bin/vlc`.

## 🎯 Проверка что всё правильно:

```bash
# На неттопе:
systemctl status ktv-daemon   # Должен быть active (running)
ss -tlnp | grep 8888          # Должен показать ktv-daemon
ls -la ~/02/09/               # Должны быть папки с временем
```

## 📞 Если проблемы:

1. Проверьте логи daemon:
   ```bash
   sudo journalctl -u ktv-daemon -n 100 --no-pager
   ```

2. Из Windows в OperatorKTV:
   - Меню → Инструменты → **Логи daemon**

3. Проверьте что VLC установлен:
   ```bash
   which vlc
   vlc --version
   ```
