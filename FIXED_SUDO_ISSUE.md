# Исправление проблемы с sudo при установке

## Проблема
При попытке установить KTV Daemon на удалённую систему возникала ошибка:
```
Installation failed with exit code 1:
sudo: a terminal is required to read the password; either use the -S option to read from standard input or configure an askpass helper
sudo: a password is required
```

## Что было исправлено

### 1. SSH Client (`operator_ktv/network/ssh_client.py`)
- ✅ Добавлено сохранение пароля при подключении
- ✅ Улучшен метод `execute_command()` для поддержки sudo с паролем через stdin
- ✅ Используется `sudo -S` для чтения пароля из stdin вместо интерактивного запроса
- ✅ Добавлена опция `get_pty=True` для корректной работы sudo
- ✅ Увеличен таймаут для длительных операций (до 300s по умолчанию)
- ✅ Пароль очищается из памяти при отключении для безопасности

### 2. Package Deployer (`operator_ktv/installer/deploy_package.py`)
- ✅ Изменён вызов установочного скрипта на использование `sudo=True`
- ✅ Пароль передаётся автоматически из SSH соединения
- ✅ Увеличен таймаут установки до 600 секунд (10 минут)

### 3. Документация
- ✅ Добавлен раздел в `DEBUGGING.md` о проблеме sudo и её решении
- ✅ Обновлён `LINUX_SETUP.md` с инструкциями по настройке sudo прав

## Как это работает сейчас

1. Вы подключаетесь к удалённой системе через SSH с паролем
2. Приложение сохраняет ваш пароль (в памяти, безопасно)
3. При установке софта используется команда: `sudo -S bash install.sh`
4. Пароль передаётся через stdin: `stdin.write(password + '\n')`
5. Установка выполняется с правами root без интерактивного запроса пароля

## Требования

Ваш SSH пользователь **должен иметь sudo права**.

Проверить:
```bash
groups
# Должно показать: sudo
```

Если нет, добавить (требуется admin доступ):
```bash
sudo usermod -aG sudo your_username
# Выйдите и войдите снова
```

## Использование

Просто используйте OperatorKTV как обычно:
1. Подключитесь к удалённой системе (Menu -> Connect)
2. Нажмите "Install Software"
3. Введите пароль SSH пользователя (если ещё не подключены)
4. Установка произойдёт автоматически с использованием sudo

Никаких дополнительных действий не требуется!

## Безопасность

- Пароль хранится только в памяти процесса
- Пароль очищается при отключении SSH
- Пароль НЕ записывается в логи
- Пароль НЕ сохраняется на диск

## Дополнительная информация

```cmd
python operator_ktv/main.py --debug
```

И проверьте логи:

```cmd
python view_logs.py -n 200
```
